### 关于 L3-诚 的关键洞察

「读懂」和「写出来」之间有一条山谷，认知科学叫「**流畅性错觉（fluency illusion）**」——读源码时会觉得"这我懂了"，真让你从零写就卡住。

M1 我们**不会跳过 ReAct 复刻**。写完那一刻你才真正拿到 L4 认证——不是自封的，是能写出来的。

### 业务金矿组合

三个方向都是 agent 的**杀手级应用场景**：

| 业务方向 | 为什么是 agent 天生适合的场景 |
|---|---|
| **归因** | 需要跨多数据源判断（adjust/appflyser/内部埋点），典型的"多工具调用"；数据异常判断人力昂贵 |
| **内部 SaaS 需求** | 需求分类 + 优先级判断 + 初诊回复；24×7 响应 |
| **线上运维/告警/日志** 🔥 | agent 最佳落地场景之一；Log → 根因 是 planner + tool use 的教科书应用；对标 PagerDuty / Datadog Watchdog / Google SRE Assistant |

**推荐用作学习路径的真实案例**：**运维告警助手** —— 数据可脱敏、边界清晰、失败代价低、成功价值高。

## 🧩 每周固定节奏

```
📖 输入 (40%, 4h)
  ├─ Mon-Wed 睡前 30-60 min × 3 = 通读教材/论文/源码
  └─ 目标：5 张 Anki 卡片，每张 ≤ 1 分钟

🔨 手动 (50%, 5h)
  ├─ 周中挑 1-2 晚（1-2h）写代码/画架构图
  └─ 周末大块 3h → 一个完整迭代

📝 输出 (10%, 1h)
  └─ 周日 30-60 min：更新 progress.md + 主动回忆自测
```

**🔴 硬约束**：哪怕这周只做了 3 小时，**周日的自测必须做** —— 自测是遗忘曲线的核心武器，不能省。
