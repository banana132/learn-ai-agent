# M1-W1-3: CoT vs ReAct vs Plan-and-Execute

> 日期: 2026-07-16
> 状态: ✅ PASS
> 模式: 学习阶段 Phase 3

---

## 核心概念

Agent 推理的三种模式，区别在于**推理和行动的顺序关系**：

| | CoT | ReAct | Plan-and-Execute |
|---|---|---|---|
| 工具调用 | 无 | 每轮交错 | 批量一次性 |
| 推理时机 | 全部在前 | 穿插进行 | 全部在前 |
| 依赖链 | 无外部依赖 | 强依赖 | 步骤独立 |
| 适用场景 | 纯推理题 | 探索性任务 | 确定性批量任务 |

**判断公式:** 不是任务类型决定模式，是**信息需求的结构**决定模式。

---

## ⭐ 用户生成的类比 (永久教学资产)

### 归因场景 · ReAct
> "先判断游戏是手游还是端游，然后再走不同的归因逻辑"

→ ReAct: 步骤1的结果（游戏类型）决定步骤2走哪个分支。

### 归因场景 · Plan-and-Execute
> "已经明确了是手游，获取用户状态 + 广告点击数据 + 历史归因数据"

→ Plan-and-Execute: 三个数据源独立，可并行拉取。

---

## 关键洞察: 同一个任务，不同模式

"写文章"这个任务：
- 纯脑内知识 → CoT
- 需查固定清单（市场数据+技术趋势+产品） → Plan-and-Execute
- 探索性研究 → ReAct

**边界由信息需求结构决定，不由任务表面类型决定。**

## 🔨 刻意练习结果

场景 A: "写 AI Agent 未来文章" → CoT ✅（默认纯推理）
