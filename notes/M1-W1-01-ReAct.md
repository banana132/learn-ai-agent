## 🎯 学到的核心

### 1. ReAct 一句话本质
> ReAct = 让 LLM 从「一口气回答」变成「边想边查边答」的循环协议

### 2. 三元组精确定义
- **Thought** = LLM 主动推理输出 (决定下一步)
- **Action** = LLM 输出的结构化工具调用
- **Observation** = 工具执行的原始返回, **原样**塞回 context

### 3. 心智模型锚点：复利
```
轮次       →   1        2        3        4
context 值 →  10%      30%      70%      95%
```

## ⭐ 我今天的独创贡献 (要记住并复用)

### 广告归因类比 (讲课杀器)
> **场景**: 广告归因程序中
> - 调用服务获取设备号 → uin ← 这是 tool call
> - 拿到 uin 后关联点击数据 ← 这是 observation
> - 判断"够不够回答" ← 这是 thought

**这个类比比论文原例更好**：
- ✅ 抓到了"**链式依赖**"本质（多步依赖前一步结果）
- ✅ 锚定到我的业务土壤（广告/归因）
- ✅ 可直接给同事讲清

**复用场景**：M4 的 multi-agent、M5 的运维告警 agent 都可以用这个类比讲清。

### Type Drift 直觉迁移 ⭐⭐
> **场景**: `uin="00123"` 被加工成 `123`, 下一步查询失效

**独立推导出了 LLM Agent 世界的真实 bug 家族**：
```
API 返回: {"user_id": "007", "amount": "1.00"}
LLM 总结: "user 7 花了 1 块钱"
下一轮:  refund(user_id=7, amount=1)  → 退错人 💥
```

**术语**: 学界叫这个 **Faithfulness (忠实性)** 破坏

---

## ❌ 今天暴露的漏洞 (已修复)

### 漏洞 1: Observation vs Thought 混淆
**原始错误**: 以为 Observation 是"总结/看是否够答"的过程
**正确**: Observation 是**被动接受**的原始 tool 返回

**修复方式**: 用广告归因类比 + Type Drift 案例强化

### 漏洞 2: 边界练习方向反了
**原始错误**: 以为 result 需要 think 加工才能变 observation
**正确**: result **就是** observation, 一字不改

**修复方式**: 3 个死因 (信息损失/幻觉入口/成本翻倍) + Faithfulness 概念

## 🧠 学到的新术语（本次首次接触）

| 术语 | 含义 | 何时用 |
|---|---|---|
| **Faithfulness (忠实性)** | Observation 必须忠实反映外部真实状态 | 讨论 agent 数据可靠性时 |
| **Type Drift (类型漂移)** | LLM 加工数据导致的类型/语义偏移 | 排查 agent bug 时 |
| **Agent 幻觉复合** | 失真数据在循环中指数级放大 | 讨论为什么 agent 会跑偏 |
