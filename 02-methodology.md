## 🚨 三大铁律（Red Flags · 全程盯住）

### ⚠️ Red Flag #1：框架陷阱

**症状**：用 LangChain / LlamaIndex 抄捷径

**为什么危险**：
- LangChain 已封装 ReAct → 你无法通过手写产生深刻记忆
- 生产问题 90% 来自框架未暴露的边界 → 不懂原理调不通
- 抽象泄漏（Leaky Abstraction）—— 你以为在写 agent，其实在写 LangChain DSL

**铁律**：**M1-M2 全程禁用高阶框架，只用 SDK（Anthropic/OpenAI）**

### ⚠️ Red Flag #2：资源囤积

**症状**：收藏 30 篇 arxiv 论文/N 个 repo 再开始学

**为什么危险**：
- 囤积 = 拖延的高级伪装
- 收藏 ≠ 阅读 ≠ 理解 ≠ 会用（4 层折损）
- 恐惧驱动的收藏没有优先级

**铁律**：**同时进行的资料 ≤ 3 份**

### ⚠️ Red Flag #3：缺主动回忆

**症状**："我读了 3 遍，肯定懂了"

**为什么危险**：
- 流畅性错觉（fluency illusion）—— 熟悉 ≠ 记住
- 艾宾浩斯遗忘曲线：24h 后忘 70%
- 被动阅读的知识留存率 = 10%

**铁律**：**读完就合上，大白话讲给"未来的自己"听**

## 🎓 复利效应：读懂 vs 写出来

```
读源码次数 →      1    2    3    5    10
─────────────────────────────────────────────
以为的理解:      30%  50%  70%  85%  95%   ← 虚假曲线
真实的理解:      10%  15%  20%  25%  30%   ← 真实曲线

写代码次数 →     0    1     2     3
真实理解:       30%  70%   85%   95%
                 ↑    ↑
                读了     写了一次就跳到 70%
                10 遍
```

**这就是为什么 M1 必须手写 ReAct** —— 输出才能暴露理解的漏洞。
